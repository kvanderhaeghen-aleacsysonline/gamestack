# AGENT_NOTES — Gamestack build learnings

Append-only log. **Every agent: READ this before starting, APPEND any gotcha you hit.**
Keep entries terse (caveman style). Newest at bottom of each section.

## Conventions (follow these)
- Target framework: **net10.0** (SDK 10.0.300 installed). All projects.
- Always prefix shell commands with `rtk` (incl. inside `&&` chains). Build: `rtk dotnet build`. Test: `rtk dotnet test`.
- Layout: `src/Gamestack.Core`, `src/Gamestack.Storage.SyncedFolder`, `src/Gamestack.Infrastructure`, `src/Gamestack.Platform`, `src/Gamestack.App`, `tests/Gamestack.Tests`.
- Refs: Storage.SyncedFolder→Core; Infrastructure→Core; Platform→Core; App→Core+SyncedFolder+Infrastructure+Platform; Tests→Core+SyncedFolder+Infrastructure.
- Root namespace = project name (e.g. `Gamestack.Core.Models`).
- `Nullable` + `ImplicitUsings` enabled everywhere. Core also has `GenerateDocumentationFile=true` → XML `///` docs REQUIRED on public members (warnings else).
- MVVM: CommunityToolkit.Mvvm (`ObservableObject`, `[ObservableProperty]`, `[RelayCommand]`). UI = Avalonia 12.0.4 Fluent theme.
- DI: Microsoft.Extensions.DependencyInjection/Hosting. Local state + settings: JSON files (NO LiteDB). Images: dependency-free header reader (NO ImageSharp). Backend: synced-folder copy (NO Graph/MSAL). See pivots below.
- Core is pure: NO UI, NO platform, NO external deps at all. All storage behind `IStorageBackend`; identity behind `IAuthProvider`; persistence behind `ILocalStateStore`/`ISettingsStore`.

## Gotchas (append as found)
- Avalonia template 12.0.4 defaults App TFM to net10.0; classlibs/xunit defaulted to net9.0 — set explicitly.
- **ImageSharp v4 requires a PAID license key at build time** (build fails: "No Six Labors license found"). v2.1.x is Apache-2.0 but trips NU1902 (known moderate CVE). Decision: NO ImageSharp — Core reads image dims from headers directly via `ImageDimensionReader` (PNG/JPEG/BMP/WEBP). Core stays pure-BCL, zero deps. Don't re-add image libs without checking license + CVE.
- Core depends on NOTHING external (pure BCL). Keep it that way — cloud/UI/platform stay out.
- **BACKEND PIVOT (decided):** MVP does NOT use Graph/MSAL. Model = lean on machine's OneDrive/SharePoint sync client. App keeps working dir (edit) + treats locally-synced OneDrive folder as "remote." Push=copy file→synced folder (OneDrive uploads); Download=copy synced→working. `IStorageBackend` first impl = `Gamestack.Storage.SyncedFolder` (folder copy, zero deps). Storage.Graph project REMOVED (re-add later for old-version byte fetch / GitHub / Drive). Identity from OneDrive registry (HKCU\Software\Microsoft\OneDrive\Accounts\Business1: UserEmail/UserName) or Windows M365 user. `IAuthProvider` impl just resolves identity (IsSignedIn=true, SignIn=resolve, SignOut=noop) — no real auth. Impl = `Platform/OneDriveIdentityProvider`.
- `Microsoft.Win32.Registry` is IN-FRAMEWORK on net10 (no package needed; adding it trips NU1510). Just `using Microsoft.Win32;` + guard with `OperatingSystem.IsWindows()` / `[SupportedOSPlatform("windows")]`.
- Persistence = JSON files (dependency-free): `Infrastructure/JsonLocalStateStore` (%AppData%/Gamestack/state.json) + `JsonSettingsStore` (settings.json). Reuse `ManifestService.JsonOptions` for consistent camelCase + enum-as-string JSON.
- Projects now: Core, Storage.SyncedFolder, Infrastructure, Platform, App, Tests (Graph removed). 28 tests green.
- **Phase 4 (game linking) built:** Per top-level directory marker `gamestack.json` = `{ "gameId": "<folder name default>" }`, managed via `Core.Projects.GameLinkService` (backend-driven: ScanAsync/EnsureMarkersAsync/CountMissingAsync/Read/Write). Auto-seeded for all top-level dirs on (re)configure (`MainWindowVM.ApplyConfigurationAsync`); returning launch with missing markers → popup via `App.Services.DialogService`+`ConfirmDialog` (`CheckProjectMarkersAsync`, run from `MainWindow.Opened` so dialog has a shown owner — `App` no longer calls InitializeAsync). Editable in new **Games** tab (`GamesViewModel`/`GamesView`). VS Code extension deferred — it will read `gamestack.json` files by `gameId`. 37 tests green. NOTE: `MainWindow.Opened` now drives startup (guarded by `_started`). LIVE detection: `MainWindowVM` runs a `FileSystemWatcher` (NotifyFilters.DirectoryName, top-level only, 2s DispatcherTimer debounce, marshalled via Dispatcher.UIThread) on the synced root → new dir → same `CheckProjectMarkersAsync` prompt; rebuilt on (re)configure. `DialogService.ConfirmAsync` restores the window from tray if hidden before ShowDialog.
- **Phase 3 (Windows integration) built:** `IStartupService`/`WindowsStartupService` (HKCU Run key, applied from MainWindowVM on init + ApplyConfiguration). `Platform/ShutdownBlockReason` (P/Invoke ShutdownBlockReasonCreate/Destroy). App `Services/WindowsIntegration` owns: system **TrayIcon** (Open/Quit, `TrayIcon.SetIcons`), **close-to-tray** (Window.Closing → Cancel+Hide; `desktop.ShutdownMode=OnExplicitShutdown`; tray Quit → desktop.Shutdown), 60s `DispatcherTimer` → **end-of-day reminder** (`ToastWindow` bottom-right) + **shutdown-hold** (updates block reason every ~5min when unpushed>0). Shutdown VETO via `Avalonia.Win32.Win32Properties.AddWndProcHookCallback` returning 0 on WM_QUERYENDSESSION(0x0011) when blocking — IS referenceable through Avalonia.Desktop. Avalonia 12: `Window.SystemDecorations`→`WindowDecorations`. Shutdown-hold is best-effort/fragile, untested against a real shutdown. 34 tests green.
- **Phase 2 (review + notifications) built:** Review model = `AssetVersion.Review` (ReviewRequest: Reviewer/RequestedBy/Status/DecidedAt). `ManifestService.RequestReview` + `SubmitReviewDecision` (verdict updates status + appends Approve/RequestChanges comment). Notifications behind `Core.Notifications.INotifier` (+ `NotificationMessage`, `ReviewNotifications` builder). Channels in Infrastructure: `SlackWebhookNotifier` (HttpClient POST), `SmtpEmailNotifier` (MailKit; M365 often blocks SMTP AUTH). `WorkspaceSession.Notifiers` rebuilt from `AppSettings.Notifications` on Configure. App `ReviewService` orchestrates manifest-mutate→save→notify(best-effort). UI: Explorer "Request review" + `ReviewInboxView`/VM (approve/request-changes) + Settings Notifications section + Reviews nav. SMTP password stored plaintext in settings.json (flagged in UI). 34 tests green.
- **Phase 7 (tags + custom attributes + search) built:** Model: `AssetFile.Tags` (List<string>) + `AssetFile.Attributes` (Dictionary<string,string>); `Manifest.Tags` (vocabulary) + `Manifest.AttributeDefinitions` (`CustomAttributeDefinition{Key,Type}`, `AttributeValueType` enum Text/Number/Boolean/Date). Core `Organization/`: `FileNameTokenizer` (Tokenize splits on non-alphanumeric→lowercase; MatchTags matches vocab case-insensitive preserving casing) + `AssetSearch`/`SearchQuery`/`AssetSearchResult` (pure AND-filter over manifest by text/tags/extension/gameId/attributes; gameId compares to Manifest.GameId). `ManifestService` gained AddTagToVocabulary/AddFileTag/RemoveFileTag/AutoTagFile/DefineAttribute/SetAttribute. App: Explorer detail pane has Tags (chips + add, ✕ remove via `$parent[ItemsControl]` binding to RemoveTagCommand) + Attributes (key/value rows + Set/remove); new **Search** nav+page (`SearchViewModel`/`SearchView`, live filter on property change). Auto-tag fires in `ChangesViewModel` after a v1 push (matches name vs vocab, re-saves manifest, surfaces in Status). DI: `SearchViewModel` AddTransient. 47 tests green. NOTE: WrapPanel tag chips bind remove cmd via `{Binding $parent[ItemsControl].((vm:ExplorerViewModel)DataContext).RemoveTagCommand}`. Attribute values stored as strings, typed by definition (no per-type validation yet — lean).
- **Phase 12 (MCP server) built:** New project `src/Gamestack.Mcp` (Exe, net10.0) refs Core+Storage.SyncedFolder+Infrastructure. Deps: `ModelContextProtocol` 1.3.0 (MIT, official SDK — now STABLE not preview) + `Microsoft.Extensions.Hosting` 10.0.0. `Program.cs`: `Host.CreateApplicationBuilder` → `AddMcpServer().WithStdioServerTransport().WithToolsFromAssembly()`; **logging MUST go to stderr** (`AddConsole(o=>o.LogToStandardErrorThreshold=Trace)`) since stdio transport owns stdout for JSON-RPC. `WorkspaceManifestAccessor` (singleton, injected into tools) loads AppSettings via `JsonSettingsStore` then reads manifest via `new SyncedFolderBackend(root).ReadTextAsync(SyncEngine.ManifestPath(""))` — READ-ONLY, never writes. `AssetTools` = `[McpServerToolType]` static class; tools `[McpServerTool(Name=...)]` + `[Description]` on method & params; return JSON strings via `ManifestService.JsonOptions`. DI services (WorkspaceManifestAccessor) + CancellationToken are injected (not part of tool schema) — put them as leading non-defaulted params, tool-arg params after with defaults. 5 read-only tools: search_assets/get_asset/list_assets/list_tags/get_workspace_info. SMOKE TEST: drive via System.Diagnostics.Process w/ redirected stdio + **async** OutputDataReceived reader (sync ReadToEnd-after-exit returned empty — flush timing); send initialize+notifications/initialized+tools/list, EOF stdin → server exits. tools/list needs NO configured workspace (good smoke test). 8 projects build clean, 47 tests green. README in project = client-registration config.
- **Avalonia 12.0.4 differences from v11:** `TextBox.Watermark` → use `PlaceholderText`. `Avalonia.Data.Core.Plugins.BindingPlugins` is inaccessible (the v11 DisableAvaloniaDataAnnotationValidation snippet won't compile — just omit it). App project: template did NOT enable ImplicitUsings — enable it. App uses MS.Ext.DI; ViewLocator maps VM→View by name (`ViewModels.XxxViewModel`→`Views.XxxView`), so page Views need parameterless ctors + matching names.
